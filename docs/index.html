Field Guide No. 001

A battle mechanics index built from replay logs.

Pokemontology tracks a battle as a sequence of states, actions, and events. It publishes an OWL ontology, SHACL validation shapes, replay tooling, and import pipelines for outside data sources.

What ships here

This site is the public build of the project. It hosts the ontology, the validation shapes, and the files you need to inspect or build against the graph.

One base namespace

IRIs and downloadable bundles resolve from the same base, so the published graph stays consistent.

Built from modules

The `.ttl` files here are generated from the modular sources in the repository, not edited by hand.

Validation ships with it

The SHACL layer is published beside the ontology because the data is meant to be checked, not just read.

Boot sequence

Rebuild the ontology and refresh the published files with:

python3 -m pokemontology build

This updates build/ontology.ttl, build/shapes.ttl, and the published copies under docs/.
