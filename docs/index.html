What is published here

The GitHub Pages site is not just documentation. It is the canonical public namespace for the built ontology and the SHACL constraints used to validate replay slices, save-state data, and source ingests.

Stable namespace

Both ontology and shapes resolve under the same public Pages base, so IRIs and downloadable artifacts stay aligned.

Build-driven publication

The published TTL files are generated from modular sources in the repository, then copied into this site during the build.

Operational validation

SHACL constraints are part of the published contract, not an internal afterthought.

Quick Start

Rebuild the ontology and refresh the Pages artifacts locally with:

python3 -m pokemontology build

That updates build/ontology.ttl, build/shapes.ttl, and the published copies under docs/.
