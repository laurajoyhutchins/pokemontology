Ontology, Shapes, and Tools

A battle mechanics ontology built from actual replay data.

Pokemontology treats a battle as a sequence of explicit states, actions, and events. The project publishes an OWL ontology, a SHACL validation layer, replay tooling, and data pipelines for external sources.

What lives here

This Pages site is the published face of the project. It hosts the canonical ontology, the validation shapes, and the links you need to inspect or build against the graph.

One public namespace

Ontology IRIs and downloadable artifacts resolve from the same base, so the published graph stays coherent.

Built from source

The `.ttl` files here are generated from the modular sources in the repository, not maintained by hand.

Validation is part of the contract

The SHACL layer is published alongside the ontology because the project cares about usable, checkable data.

Quick start

Rebuild the ontology and refresh the Pages artifacts with:

python3 -m pokemontology build

This updates build/ontology.ttl, build/shapes.ttl, and the published copies under docs/.
